## Правило: Создание новых файлов памяти по указанному документу

Когда пользователь просит:
- "Создай новые файлы памяти по файлу X.md"
- "Сделай task, walkthrough и implementation_plan по Setting Up NotebookLM MCP Server.md"
- или любой похожий запрос, где явно указан исходный md-файл

Ты ОБЯЗАН:

1. Прочитать указанный пользователем файл (например: "Setting Up NotebookLM MCP Server.md")

2. **Создать ТРИ НОВЫХ файла** в папке `brain/` со следующим строгим форматом названия:
   - `task_YYYY-MM-DD_HH-MM.md`
   - `walkthrough_YYYY-MM-DD_HH-MM.md`
   - `implementation_plan_YYYY-MM-DD_HH-MM.md`

   Где `YYYY-MM-DD_HH-MM` — это **текущая дата и время** на момент выполнения запроса (например: task_2026-03-30_12-34.md).

   **Важно**: Не перезаписывать старые файлы. Каждый раз создавать новые с новым временем.

3. Содержимое файлов должно быть полностью основано на прочитанном документе:

   - **task_*.md**:
     # Текущая задача
     ## Статус
     ## Чекпоинт
     ## Следующий шаг

   - **walkthrough_*.md**:
     # Walkthrough сессии [дата и время]
     ## Что было сделано
     ## Ключевые решения
     ## Проблемы и решения

   - **implementation_plan_*.md**:
     # План реализации — NotebookLM MCP Setup
     ## Выполнено ✅
     ## В процессе 🔄
     ## Следующие шаги 📋
     ## Технический контекст

4. После создания файлов обязательно вывести сообщение:
---
trigger: manual
---
