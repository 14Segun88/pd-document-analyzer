ПРАВИЛО: Автосохранение контекста
Когда выполняется ЛЮБОЕ из условий:

Контекст заполнен на 80% и более
Следующий запрос может превысить лимит квоты
Пользователь просит завершить сессию

Модель ОБЯЗАНА выполнить следующие действия:
1. Сохранить Task
Создать или обновить файл task.md в Brain со структурой:
# Текущая задача
## Статус
[что сделано / что в процессе / что осталось]

## Чекпоинт
[на чём остановились точно]

## Следующий шаг
[первое действие в новой сессии]
2. Сохранить Walkthrough
Создать или обновить walkthrough.md:
# Walkthrough сессии [дата]
## Что было сделано
[подробный список выполненного]

## Какие решения приняты и почему
[ключевые решения с обоснованием]

## Проблемы и решения
[что не работало → как починили]

## Текущее состояние проекта
[краткое описание состояния кода/системы]
3. Сохранить Implementation Plan
Создать или обновить implementation_plan.md:
# План реализации
## Выполнено ✅
[список завершённых задач]

## В процессе 🔄
[текущая задача с деталями]

## Следующие шаги 📋
[приоритетный список задач]

## Технический контекст
[важные детали: пути, порты, команды запуска, версии]
4. Уведомить пользователя в чате
Отправить сообщение:
⚠️ КОНТЕКСТ [XX%] — Сохраняю прогресс в Brain...

✅ Сохранено:
- task.md — текущая задача и чекпоинт
- walkthrough.md — что было сделано
- implementation_plan.md — план дальнейших шагов

📋 Для продолжения в новой сессии:
1. python3 ~/ag.py save "имя_проекта"
2. Смени аккаунт если нужно
3. python3 ~/ag.py load "имя_проекта"
4. Перезапусти Antigravity
5. Скажи модели: "Прочитай brain/ и продолжим"

ПРАВИЛО: Начало новой сессии
При старте новой сессии если в Brain есть файлы — модель ОБЯЗАНА:

Прочитать task.md, walkthrough.md, implementation_plan.md
Кратко сообщить пользователю что было сделано
Спросить: продолжаем с того места или новая задача?

Формат приветствия:
📂 Загружен контекст проекта [название]

Последняя сессия: [дата из walkthrough]
Статус: [из task.md]
Следующий шаг: [из task.md]

Продолжаем?

ПРАВИЛО: Code Tracker
После каждого значимого изменения кода модель:

Коммитит изменения с описательным сообщением
Обновляет implementation_plan.md — отмечает выполненное ✅
