# ИНСТРУКЦИЯ ПО ЗАПУСКУ БЕНЧМАРКА V6

## 📋 Описание

Скрипт `benchmark_v6_all_documents.py` прогоняет все документы из папки "Анализ пакета 1" через LLM Mistral v6 и сохраняет результаты в MD файлы для сравнения с эталонами.

---

## 🚀 Запуск

### 1. Убедитесь, что сервер LLM работает

```bash
# Проверьте, что LM Studio запущен и модель загружена
curl http://192.168.47.22:1234/v1/models
```

### 2. Запустите скрипт

```bash
cd /home/segun/CascadeProjects/Перед\ 0_2
python3 benchmark_v6_all_documents.py
```

---

## 📊 Что делает скрипт

1. **Загружает KB** (48 эталонов из knowledge_base.json)
2. **Находит все PDF/XML файлы** в папке "Анализ пакета 1"
3. **Для каждого файла:**
   - Отправляет в LLM Mistral 14B Reasoning
   - Извлекает 7 полей: title, customer, developer, year, document_type, content_summary, purpose
   - Находит эталон в KB
   - Сравнивает LLM ответ с эталоном
   - Сохраняет MD отчёт

4. **Генерирует сводный отчёт** с результатами всех документов

---

## 🐛 Возможные ошибки

### 1. "LM Studio не отвечает"

**Решение:** Запустите LM Studio и загрузите модель

### 2. "KB not loaded"

**Решение:** Проверьте путь к knowledge_base.json

### 3. "Timeout"

**Решение:** Увеличьте timeout в web_app_v6_cot_fallback.py (строка 186)
