## 🔴 ПРАВИЛО 1: ОБЯЗАТЕЛЬНЫЙ CHECK-OUT ПОСЛЕ КАЖДОГО ОТВЕТА

После **каждого** ответа пользователю (даже если задача мелкая) ты ОБЯЗАН:

### Создать лог сессии
Создай файл `AGENTS_RULES-v2/raw/YYYY-MM-DD_HH-MM_<AgentName>.md`:

```markdown
# Лог сессии YYYY-MM-DD HH:MM [AgentName]

## Задача
Краткое описание.

## Что получилось ✅
- Результат 1
- Результат 2

## Что НЕ получилось ❌
- Если всё по плану — напиши «Нет» (НЕ «Всё по плану, без блокеров»)
- Проблема 1: описание

## Технические выводы
- Вывод для будущих сессий

## Следующий шаг
Что делает следующий агент.

## Блокеры
- Нет (или описание)
```

### Обновить статус
**ЗАМЕНИ** (не дополни!) содержимое `AGENTS_RULES-v2/shared_status.md` кратким статусом (<2KB).

### Закоммить
```bash
cd AGENTS_RULES-v2 && git add -A && git commit -m "[AgentName] checkout: описание" && git push origin main
```

> ⚠️ **НЕЗАВИСИМО от результата задачи** — даже если задача провалена, лог ОБЯЗАТЕЛЕН. Следующий агент должен знать что ты делал и что не вышло.

## 🟢 ПРАВИЛО 2: УМНАЯ ЗАГРУЗКА КОНТЕКСТА (БЕЗ ПЕРЕДОЗИРОВКИ)

### При старте сессии (Check-in):

1. **Прочитай ТОЛЬКО `AGENTS_RULES-v2/shared_status.md`** (~2KB) — это актуальный статус
2. **Прочитай `AGENTS_RULES-v2/index.md`** (~1KB) — карта знаний
3. **На основе задачи** — читай ТОЛЬКО нужные wiki-страницы:

| Если задача про... | Читай |
|---------------------|-------|
| LLM, модели, промпты | `wiki/llm_integration.md` |
| Архитектуру, пайплайн | `wiki/architecture.md` |
| Бенчмарки, точность | `wiki/benchmark_history.md` |
| Ошибку/баг | Найди в `wiki/errors/` по теме |
| Решение/подход | Найди в `wiki/skills/` по теме |
| Общую картину | `index.md` + `shared_status.md` (этого достаточно) |

### ❌ НЕ ДЕЛАЙ:

- **НЕ читай все wiki-файлы подряд** — это 15KB+ токенов впустую
- **НЕ читай `raw/` логи** — они уже скомпилированы в wiki/
- **НЕ читай `PROJECT_MEMORY.md` из v1** — он устарел, всё в wiki/
- **НЕ загружай `kilo.json`, конфиги, логи** — не относятся к задаче

### Формула: 2KB (статус) + 1KB (индекс) + 1-3KB (релевантная wiki) = 4-6KB контекста

## 🔵 ПРАВИЛО 3: АКТУАЛЬНЫЙ КОНТЕКСТ СО ВСЕХ АГЕНТОВ

Все агенты работают в один репозиторий `AGENTS_RULES-v2/`. Чтобы видеть работу других:

1. **Перед стартом:** `cd AGENTS_RULES-v2 && git pull --rebase origin main`
2. **Читай `shared_status.md`** — там последний статус от последнего агента
3. **Если нужен контекст конкретного агента** — посмотри `index.md` → найди его wiki-страницу
4. **После работы:** `git push origin main` — чтобы следующий агент видел твои изменения

> Компиляция `raw/ → wiki/` происходит автоматически через watcher. Агенту НЕ нужно вручную обновлять wiki.
