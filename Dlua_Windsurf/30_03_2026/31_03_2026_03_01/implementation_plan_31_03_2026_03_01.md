# План исправления ошибок извлечения для Пакета 2

## Обзор проблемы
Модель Llama 70B работает, но возвращала слишком краткие ответы, ошибалась в `document_type`, `year`, `developer` и не понимала слово "шифр".
Причина: жёсткие правила в `PROMPT_SYSTEM` и неправильная логика подбора эталонов из `knowledge_base.json`.

## Выполненные изменения ✅

### [MODIFY] [web_app_v5.py](file:///home/segun/CascadeProjects/Перед%200_2/Перед%200/project_analyzer/web_app_v5.py)

#### 1. Semantic Matching для подбора эталонов ✅
Реализована система баллов, сопоставляющая имя загруженного файла с полем `document_kind` каждого эталона в `knowledge_base.json`. Теперь модель видит релевантный эталон, а не случайные первые два.

#### 2. PROMPT_SYSTEM — Раунд 1 ✅
- Убран шаблон `"Текстовый и графический документ (тип) – часть ПД/РД"`.
- Запрещены фамилии в `developer`.
- `content_summary` обязан быть маркированным списком.
- Поиск "ШИФРА" в `title`.

#### 3. PROMPT_SYSTEM — Раунд 2 ✅
- `year`: поиск даты ВЫДАЧИ, а не исторических дат.
- `document_type`: СУРОВЫЙ ЗАПРЕТ на фразу "часть ПД/РД".
- `content_summary`: запрет на координаты (X, Y) и дублирование шапки.
- `purpose`: живые примеры вместо "Для строительства".

#### 4. Исправление JSON Parse Error ✅
Добавлен `strict=False` в `json.loads()` для обработки многострочных ответов LLM.
