# Долгосрочная память (MEMORY.md)

## Контекст проекта
- Текущий проект: Интеллектуальный анализатор документов для строительных проектов (MБОУ СОШ на 1100 мест).
- Стек: Python, Flask, PyMuPDF, LLM (LM Studio), Vue 3 на фронтенде.
- Архитектура: In-Context Learning (Few-Shot), инъекция примеров из `knowledge_base.json` в системный промпт LLM для сохранения структуры ответа без Fine-Tuning.

## Успешные решения и Skills
- [Инъекция контекста через knowledge_base]
  Описание: Выделен отдельный файл knowledge_base.json с эталонными ответами, который пробрасывается в вызов LLM, что исключает ошибки структуры (Zero-Shot -> Few-Shot).
  Дата: 2026-03-30
  
- [Универсальный системный промпт без фейковых диалогов]
  Описание: Выяснилось, что при передаче грязного текста из документов нельзя использовать жесткий Few-Shot с искусственными диалогами ('Извлеки из: Документ Х...'). Модель начинает лениться и искать идеальные предложения, которых нет в грязном тексте. Вместо этого эталоны нужно подавать просто внутри `<system>` как справочник 'ПРИМЕРЫ ЖЕЛАЕМЫХ ОТВЕТОВ'.
  Пример использования: `messages = [{"role": "system", "content": PROMPT_SYSTEM + examples_text}, {"role": "user", "content": text}]`
  Дата: 2026-03-30

- [Ограничение входа для защиты окна контекста (Context Window Exceeded)]
  Описание: Модели Llama 3 70B и другие часто падают с ошибкой `'The number of tokens... n_keep >= n_ctx: 4096'`, если им отправляется слишком много текста. Ограничение входящего текста `text[:6000]` перед отправкой надежно решает эту проблему без ручного увеличения окна в самом LM Studio.
  Дата: 2026-03-30

## Ошибки и ограничения
- [Галлюцинации от жесткого One-Shot примера (Model Overfitting)]
  Проблема: При добавлении в промпт всего одного фейкового примера `{customer: ООО Ромашка}` перед реальным запросом, мощные instruct-модели могут впасть в ленивое состояние и начать возвращать `ООО Ромашка` на ЛЮБЫЕ новые документы, если текст кажется им слишком сложным для парсинга.
  Как делать правильно: Исключать любые выдуманные имена/названия из истории сообщений `role: user` / `role: assistant`. Загружать только подлинные примеры из базы в `system`.
  Дата: 2026-03-30

- [Model Unloaded (Ответ LM Studio)]
  Проблема: При указании жесткого имени модели (например, meta-llama-3.3-70b-instruct) LM Studio возвращает ошибку Model unloaded или Operation canceled, если модель в данный момент не загружена в ОЗУ.
  Как делать правильно: Использовать универсальный синоним `"local-model"` в параметре `model_name`.
  Дата: 2026-03-30 13:08

- [Ответы на английском языке из-за англоязычных ключей]
  Проблема: При указании правила "верни JSON-словарь с ключами на английском", LLM может ошибочно перевести и сами значения (values) на английский язык.
  Как делать правильно: Жестко прописывать правило: "ВСЕ ЗНАЧЕНИЯ в JSON должны быть СТРОГО НА РУССКОМ ЯЗЫКЕ. Никакого перевода на английский!".
  Дата: 2026-03-30

### Ошибки и ограничения
- [Жесткие инструкции промпта ломают Few-Shot]
  Проблема: Модель писала слишком короткое `content_summary` и неправильный `document_type` (копировала шаблон "Текстовый и графический документ"), игнорируя эталонные ответы для ЕГРН/ГПЗУ/ТУ. Она также выписывала фамилии людей вместо компаний.
  Как делать правильно: Ни в коем случае не зашивать жесткие текстовые шаблоны (например, "Для PDF пиши вот так...") в системный промпт для полей `document_type` или `content_summary`. Требовать маркированный список явным образом ("ТРЕБУЕТСЯ ДЕТАЛЬНЫЙ МАРКИРОВАННЫЙ СПИСОК"), запрещать писать имена людей ("Нам нужно ЮРИДИЧЕСКОЕ ЛИЦО"). 
  Пример использования: Использование слова "ШИФР" добавлено в prompt title. Логика `web_app_v5.py` перестроена для умного подбора эталонов (Semantic Matching) по имени файла, чтобы модель видела эталон, подходящий именно к текущему документу.
  Дата: 2026-03-30
